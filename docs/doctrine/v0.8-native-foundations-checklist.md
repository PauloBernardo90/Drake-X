# Drake-X v0.8 Native Foundations Checklist

Operational implementation checklist for the v0.8 phase.

- Date: 2026-04-11
- Scope: Drake-X v0.8
- Status: Active implementation guide

## Objective

Deliver the first native-binary domain for Drake-X without drifting into
exploit tooling.

v0.8 is complete when Drake-X can:

- ingest a PE sample as a first-class artifact
- detect and route binary format correctly
- parse PE headers, sections, imports, exports, and resources
- assess section anomalies and protection status
- perform bounded disassembly at function scope
- produce evidence-backed PE findings and reports
- expose the workflow via `drake pe analyze`

v0.8 does not include:

- ROP reconstruction
- broad memory modeling
- debugger-first workflows
- shellcode operational decoding
- exploit simulation

## Definition of Done for v0.8

v0.8 is done when all of the following are true:

- a `.exe` or `.dll` can be ingested as a PE sample
- the platform detects the format and routes analysis correctly
- headers, sections, imports, exports, and resources are parsed
- section anomalies and protection status are assessed
- bounded disassembly is available
- a PE technical report is generated
- PE-native evidence fits the shared platform evidence model
- the workflow is exposed through `drake pe analyze`

## Explicit Non-Goals

The following remain out of scope for v0.8:

- ROP reconstruction
- broad memory models
- debugger integration as a primary workflow
- exploit simulation
- operational payload recovery
- exploit-chain optimization

## Review Rule

Any v0.8 proposal that introduces exploit-operational behavior,
unbounded instruction storage, or debugger-replacement UX should be
rejected or deferred to a separate design review.
